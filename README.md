# Google Play Store Data Cleaning

This project cleans and prepares Google Play Store app data and user review data for later analysis and visualization.

The main workflow lives in [main.ipynb](main.ipynb), where the raw datasets are explored, cleaned, transformed, and exported into the `clean/` folder.

## Project Structure

- `main.ipynb` - notebook used for exploration, cleaning, and export
- `raw/` - original source CSV files
- `clean/` - cleaned CSV files generated by the notebook
- `src/` - presentation and edit mode assets
- `Power Query for cleaned data.pbix` - Power BI file for working with the cleaned data
- `Power Query Presentation.pdf` - presentation output for the project

## Datasets

The notebook works with two datasets:

- `Play Store Data.csv`
- `User Reviews.csv`

Both are stored in the `raw/` folder and are written back to the `clean/` folder after processing.

## Cleaning Workflow

The notebook performs the following steps:

1. Load the raw CSV files into pandas DataFrames.
2. Inspect shapes, missing values, duplicates, and summary statistics.
3. Clean key Play Store fields such as `Installs`, `Price`, `Size`, `Reviews`, `Last Updated`, and `Rating`.
4. Handle missing values using category-based summary values where appropriate.
5. Remove invalid rows and duplicate records.
6. Export the cleaned datasets to `clean/Play Store Data.csv` and `clean/User Reviews.csv`.

## Key Transformations

- Converted `Installs` into integers.
- Converted `Price` into numeric values.
- Standardized `Size` into numeric megabytes.
- Converted `Reviews` into numeric counts.
- Parsed `Last Updated` as dates and extracted a `Year` column.
- Filled missing `Size` and `Rating` values using category-level averages and medians.
- Removed duplicates from both datasets.

## Reproducing the Cleaned Data

To regenerate the cleaned CSV files:

1. Open [main.ipynb](main.ipynb).
2. Run the cells from top to bottom.
3. Confirm that the files in `clean/` are updated.

## Notes

The notebook is designed as a data preparation step before analysis or dashboard work. The cleaned files in `clean/` can be used directly in Power BI, Python, or other visualization tools.
